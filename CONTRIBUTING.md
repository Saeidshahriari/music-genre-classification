# Contributing

## Workflow
1. Create a branch from `main`: `feature/<name>` or `fix/<name>`.
2. Make changes, commit with clear messages (see below).
3. Open a Pull Request, get one approval, merge.

## Commit messages
Format: `<type>: <short description>`
Types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`.

Example: `feat: add MFCC feature extraction`

## Before pushing
- Run `make test`
- No data files, no secrets
